## 🔐 SISTEMA DE PERMISOS GRANULARES

### Permisos disponibles
- `crear_auditoria`
- `ejecutar_auditoria`
- `ver_sus_auditorias`
- `ver_todas_auditorias`
- `exportar_reportes`
- `gestionar_checklist`
- `gestionar_usuarios`
- `modificar_permisos`
- `ver_dashboard`
- `crear_sitio`
- `editar_sitio`

### Funcionamiento
- Cada rol tiene permisos base definidos en `src/types/index.ts` → `ROLES_PERMISOS`
- El admin puede sobrescribir permisos por usuario (overrides)
- Los overrides pueden tener fecha de vencimiento
- Si la fecha vence, los overrides se ignoran y vuelven los del rol base
- El historial de cambios se guarda en colección `permisosHistorial`

### Funciones clave
- `tienePermiso(user, permiso)` → Verifica permiso (considera overrides y vencimiento)
- `getPermisosEfectivos(user)` → Devuelve todos los permisos efectivos
- `overridesVencidos(user)` → Verifica si los overrides expiraron
- `getOrigenPermisos(user)` → Devuelve origen de cada permiso ('rol' | 'override')

### Hook disponible
- `usePermissions()` en `src/hooks/usePermissions.ts`
  - `.can('permiso')` → boolean
  - `.isAdmin`, `.canAuditar`, `.canGestionar`, etc.

## 🔄 FLUJO DE PERMISOS

1. Usuario hace login → AuthContext carga datos de Firestore
2. Datos incluyen: `permisosOverride`, `overrideActivo`, `fechaVencimientoOverrides`
3. `tienePermiso()` verifica:
   - Si es admin → true
   - Si overrideActivo y no venció y permiso definido → usa override
   - Si no → usa ROLES_PERMISOS[rol]
4. `App.tsx` usa permisos para mostrar/ocultar pestañas
5. `AppContext` expone flags granulares para cualquier componente
